RISC 机器说明

RISC = 生产级 Agent / 机器人身体的四个系统

一个真正能上生产的 Agent，不能只有大脑。它要能持续运行，要能判断和行动，要能抵御错误、攻击和投毒，也要能进入真实世界的协作网络。

行业已经交付了一颗出色的大脑，但生产级 Agent 还需要机体、免疫和社会。ALUX 构建的，就是这台完整机器。
R｜强韧 / 机体容错、持久执行、故障切换、水平扩展。没有强韧机体，一次崩溃就会带走全部工作。
I｜智能 / 大脑推理循环、记忆、工具调用、任务编排。这是今天所有 Agent 框架最拥挤、也最成熟的竞争层。
S｜安全 / 免疫对象能力、策略约束、回滚机制、审计链路。没有安全免疫，一条被投毒的指令就可能造成真实损害。
C｜连接 / 社会跨公司授权、中立基底、会话类型、协作边界。没有连接网络，每家公司的 Agent 都只能困在自己的孤岛里。

01Google Gemini CLI美国/全球开源2026-07-18 / 2026-07-19 观察官方预发布

Gemini CLI 夜版把 macOS 沙箱改为默认拒绝，并同时收束无限 ReAct 与提示注入循环

发生了什么：Google 在同一 nightly 中把 macOS permissive Seatbelt profile 对齐到 deny-default，并修复可能由提示注入或 ReAct 失控触发的无限循环。

对 ALUX 的关系：这条消息把两个常被分开的风险放到同一执行边界：Agent 既可能在认知循环里失控，也可能借沙箱例外把错误变成真实动作。ALUX 应让默认拒绝、循环预算、能力范围和终止原因进入同一份运行收据。

可行动建议与产出物：建议形成 Coding Agent Fail-Closed Envelope，统一 sandbox profile、loop budget、tool capability、stop reason 与 replay receipt。 可沉淀为：Coding Agent Fail-Closed Envelope v0。

RISC：S 主 · 安全 / 免疫R 辅 · 强韧 / 机体

这条消息主要动到机器人的安全/免疫：默认拒绝和提示注入循环控制决定危险动作能否越过边界；强韧/机体为次维度，因为无限循环与错误终止直接消耗生产资源并影响恢复。

隔离边界是macOS permissive Seatbelt profiles 改为 deny-default，未授权能力默认不可用。
策略审批部分无限 ReAct 与提示注入循环被显式缓解，但发布说明未给出风险级别审批矩阵。
合作/学习窗口

可将 Gemini CLI 作为编码 Agent 适配对象，验证 ALUX 能否在模型循环之外施加不可降级的能力与预算门。 学习 deny-default profile、循环上限和注入触发的停止条件如何进入开发者工具。

风险边界

这是 nightly 预发布；修复说明不等于所有注入路径或沙箱逃逸已被证明关闭。

今天的新变化

7 月 18 日首次把 macOS deny-default 沙箱、无限 ReAct 循环和提示注入循环修复放进同一 nightly。

02Alibaba Qwen Code中国/全球开源2026-07-18 / 2026-07-19 观察官方发布

对 ALUX 的关系：编码 Agent 的“会话”正在从聊天记录变成组织资产：它有工作区归属、归档、技能集合、启动轨迹和可查询状态。ALUX 可把这些映射为有所有者、能力版本和恢复边界的会话类型。

可行动建议与产出物：建议沉淀 Workspace Session Type v0，定义 owner、skill set、archive state、startup trace、approval state 与 recovery cursor。 可沉淀为：Workspace Session Type v0。

RISC：C 主 · 连接 / 社会S 辅 · 安全 / 免疫

这条消息主要动到机器人的连接/社会：工作区级会话、归档与技能管理让 Agent 进入组织协作状态；安全/免疫为次维度，因为会话归属与技能可用范围决定谁能继续执行。

会话类型是稳定版提供工作区会话聚合、归档导出与启动追踪，显式区分会话生命周期。
生态连接部分技能管理成为产品页面，但发布说明未证明跨厂商连接器的统一治理。
合作/学习窗口

适合探索 Qwen Code session/skill metadata 到 ALUX capability/session schema 的适配。 学习稳定版如何暴露 workspace-level session state、归档与技能生命周期。

风险边界

稳定版说明列出功能，但未证明跨公司委派、中立执行或完整权限形式化。

今天的新变化

相较 7 月 18 日日报覆盖的 nightly，0.19.12 已转为稳定版，并新增工作区会话聚合、技能管理和归档导出等正式入口。

今天的新变化

7 月 17 日官方文章首次把 Amazon Quick 的销售 Agent 从线索判断一路展示到跨工具行动、团队共享和 CRM 回写。

对 ALUX 的关系：观测产品正从固定仪表盘转向“对任意执行事件即时提问和生成视图”。ALUX 待构建的可观测层可以借鉴这种交互，但底层数据必须来自稳定运行事件、策略裁决与重放收据，而不是只做漂亮图表。

可行动建议与产出物：建议形成 Replay-Aware Event Schema，把 run、capability、policy、effect、checkpoint 与 replay status 暴露给观测工具。 可沉淀为：Replay-Aware Event Schema v0。

RISC：I 主 · 智能 / 大脑R 辅 · 强韧 / 机体

工具编排是Agent 可从 dashboard widget 入口组织事件数据与可视化工具。

风险边界

图表和观测轨迹不等于故障恢复、回滚或确定性重放；Agent 生成面板也可能放大错误查询。

今天的新变化

相较 7 月 15 日日报的 monitors 与审计日志，3.221.0 新增事件视图图表化和 Agent 驱动的面板搭建入口。

06OpenAI Codex美国/全球2026-07-18 / 2026-07-19 观察官方发布

OpenAI Codex 0.144.6 修正 GPT-5.6 系列上下文元数据，模型能力也需要运行时契约对齐

发生了什么：OpenAI Codex 0.144.6 刷新 GPT-5.6 Sol、Terra、Luna 的 bundled instructions，并把三者上下文窗口元数据纠正为 272,000 tokens。

对 ALUX 的关系：这是一条小而重要的运行契约信号：Agent 调度器若误判上下文边界，会影响截断、成本、任务拆分和恢复。ALUX 应把模型能力元数据视为版本化环境输入，进入运行记录而非静态常量。

可行动建议与产出物：适合补一份 Model Environment Receipt，记录 model id、instruction bundle、context limit、tool schema 与 client version。 可沉淀为：Model Environment Receipt v0。

RISC：I 主 · 智能 / 大脑R 辅 · 强韧 / 机体

这条消息主要动到机器人的智能/大脑：模型指令与上下文窗口决定推理可用空间；强韧/机体为次维度，因为错误元数据会让长流程在截断、成本和恢复点上失真。

推理深度部分客户端纠正 272,000-token context window，但未发布新的推理评测。
记忆使用部分更准确的上下文元数据影响历史内容装载，但不等于持久记忆。
合作/学习窗口

可将 Codex/model metadata 接入 ALUX runtime environment record，支持事后重放与成本解释。 学习模型说明与 context-window metadata 如何随客户端版本被精确固定。

风险边界

版本只修正元数据与指令，不代表模型推理能力、可靠性或安全性发生实质升级。

今天的新变化

0.144.6 首次明确修正 GPT-5.6 Sol、Terra、Luna 的 bundled metadata 与 272,000-token context window。

融资 / 合作窗口

最直接窗口：Gemini CLI 与 Qwen Code 可作为编码 Agent 适配样本；Amazon Quick、Salesforce、Outlook、Slack 和 CrewAI Skills 则提供企业能力与技能供应链入口。ALUX 的切入点是能力合同、会话类型与效果收据。
融资叙事窗口：今天没有值得放大的新融资金额。更有力的证据是产品层开始同时为 deny-default、组织会话、技能分发与跨工具行动付出工程成本，说明“完整机器”正在成为真实预算项。

技术 / 产品启发

优先产品：Agent Responsibility Envelope v0。字段建议包括 execution_domain、loop_budget、session_owner、skill_source、capability_grant、approval_state、effect_receipt、model_environment 与 replay_status。
优先 Demo：让 Qwen Code 或 Gemini CLI 从企业工作区领取销售任务，通过 CrewAI Skill 调用 Quick/CRM；中途触发提示注入与循环超限，展示 ALUX 如何默认拒绝、保留会话归属、限制外部效果并从同一长交易恢复。

风险边界

ALUX 不能被说成已经完整交付智能体平台。当前底层 TVM 已具备并发、持久化执行、能力安全、运行记录和逐比特重放审计等关键基础；智能体产品层、可观测性、仪表盘、追踪与评估工具仍是待构建和融资重点。也不要说 TVM 让 LLM 本身确定性。准确说，TVM 记录模型输出和运行环境输入，使编排、权限、状态转移和审计可重放、可验证。deny-default 沙箱、循环修复、会话聚合、技能仓库、观测图表和模型元数据，都不能自动证明跨步骤原子回滚、对象能力或中立跨公司协作。
