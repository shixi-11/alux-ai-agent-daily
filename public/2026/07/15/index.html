RISC 机器说明

RISC = 生产级 Agent / 机器人身体的四个系统

一个真正能上生产的 Agent，不能只有大脑。它要能持续运行，要能判断和行动，要能抵御错误、攻击和投毒，也要能进入真实世界的协作网络。

行业已经交付了一颗出色的大脑，但生产级 Agent 还需要机体、免疫和社会。ALUX 构建的，就是这台完整机器。
R｜强韧 / 机体容错、持久执行、故障切换、水平扩展。没有强韧机体，一次崩溃就会带走全部工作。
I｜智能 / 大脑推理循环、记忆、工具调用、任务编排。这是今天所有 Agent 框架最拥挤、也最成熟的竞争层。
S｜安全 / 免疫对象能力、策略约束、回滚机制、审计链路。没有安全免疫，一条被投毒的指令就可能造成真实损害。
C｜连接 / 社会跨公司授权、中立基底、会话类型、协作边界。没有连接网络，每家公司的 Agent 都只能困在自己的孤岛里。

01AWS WAF / Bedrock AgentCore美国/全球2026-07-14 / 2026-07-15 观察官方安全博客

AWS WAF 用 HTTP 消息签名识别合法 Agent 流量，Agent 身份开始进入网络执行边界

发生了什么：AWS 详解 Web Bot Authentication：Agent 请求按 RFC 9421 使用非对称密钥签名，WAF 从公开目录取公钥并标记 verified、invalid、expired 或 unknown_bot，再据此放行、限速或阻断。AWS 明确指出，共享 IP 的 AgentCore 多租户环境已让 IP 白名单和 User-Agent 失去可信度。

对 ALUX 的关系：这是 Agent 身份从应用账号下沉到每次网络动作的强信号。ALUX 可把签名主体、能力授予、工具动作、策略裁决和长交易状态合并成一条责任链；差异不只是“谁发来请求”，还要证明它当时获准做什么、做后能否重放。

可行动建议与产出物：建议形成 Signed Agent Capability Envelope v0，把 agent key、issuer、scope、expiry、policy verdict、tool action 与 replay evidence 放进同一结构。 可沉淀为：Signed Agent Capability Envelope v0。

RISC：S 主 · 安全 / 免疫C 辅 · 连接 / 社会

这条消息主要动到机器人的安全/免疫：每次 Agent 网络请求开始用加密身份和策略标签防伪、防冒充；连接/社会是次维度，因为开放签名标准让不同服务可以共同识别机器主体。

能力对象部分请求携带可验证签名，但签名身份证明尚不等于细粒度、可衰减能力对象。
策略审批是WAF 将 verified、invalid、expired、unknown_bot 标签直接用于允许、阻断或限速规则。
合作/学习窗口

适合连接云安全、WAF、Agent gateway 与企业身份团队，讨论签名 Agent 身份如何映射到可衰减能力对象。 学习使用开放标准表达机器身份，并把身份验证结果变成执行策略输入。

风险边界

WBA 证明请求来源，不证明请求意图安全、权限最小或后续动作可回滚；WAF 也不是跨公司中立运行时。

今天的新变化

7 月 14 日 AWS 首次把共享 IP 的 AgentCore 多租户场景与 WAF 加密 Agent 身份验证放进同一实施指南。

融资 / 合作窗口

最直接窗口：云安全、WAF、Agent gateway、身份治理、科研 Agent、观测评估和金融合规团队。共同问题是：机器主体获准行动后，如何在重启、handoff 和长时间执行中维持同一责任链。
融资叙事窗口：Hadrius 把 2700 万美元融资与 audit-ready Agent 基础设施绑定；ALUX 应强调普通日志只能描述过去，可重放运行时才能验证权限、状态转移和恢复结果。

技术 / 产品启发

优先产品：Signed Agent Capability Envelope v0。字段包括 agentKey、issuer、scope、expiry、policyVersion、action、checkpoint、resumeEpoch 与 replayVerdict。
优先 Demo：让一个签名 Agent 启动十小时级研究长交易，中途授权过期并重启 worker；展示 ALUX 如何拒绝旧能力、保留 checkpoint、重新批准后续跑并重放全链路。

风险边界

ALUX 不能被说成已经完整交付智能体平台。准确说，当前底层 TVM 已具备并发、持久化执行、能力安全、运行记录和逐比特重放审计等关键基础；智能体产品层、可观测性、仪表盘、追踪与评估工具仍是待构建和融资重点。

也不要说 TVM 让 LLM 本身确定性。准确说，TVM 记录模型输出和运行环境输入，使编排、权限、状态转移和审计可重放、可验证。AWS 的签名身份不等于能力安全；NVIDIA 的 session diary 不等于自动故障恢复；Langfuse 的 trace 与 Hadrius 的 system of record 也不等于逐比特重放。所有公开能力均按来源边界判断。
