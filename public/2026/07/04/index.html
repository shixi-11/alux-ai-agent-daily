RISC 机器说明

RISC = 生产级 Agent / 机器人身体的四个系统

一个真正能上生产的 Agent，不能只有大脑。它要能持续运行，要能判断和行动，要能抵御错误、攻击和投毒，也要能进入真实世界的协作网络。

行业已经交付了一颗出色的大脑，但生产级 Agent 还需要机体、免疫和社会。ALUX 构建的，就是这台完整机器。
R｜强韧 / 机体容错、持久执行、故障切换、水平扩展。没有强韧机体，一次崩溃就会带走全部工作。
I｜智能 / 大脑推理循环、记忆、工具调用、任务编排。这是今天所有 Agent 框架最拥挤、也最成熟的竞争层。
S｜安全 / 免疫对象能力、策略约束、回滚机制、审计链路。没有安全免疫，一条被投毒的指令就可能造成真实损害。
C｜连接 / 社会跨公司授权、中立基底、会话类型、协作边界。没有连接网络，每家公司的 Agent 都只能困在自己的孤岛里。

01Microsoft Agent Framework美国/开源2026-06-30 / 2026-07-04官方 GitHub

Microsoft Agent Framework 1.10.0 发布，把 Durable Task worker、工具审批、遥测和 Harness loop 推进到框架核心

发生了什么：MAF 1.10.0 增加 standalone Durable Task worker、Foundry session helper、工具审批默认化、AgentLoop/HarnessAgent 集成、OTel tool format 与多处 checkpoint / telemetry 修复。

对 ALUX 的关系：Microsoft 正把 agent framework 从编排工具推到可托管、可恢复、可观测的运行时。ALUX 应把它视为最接近“联邦化 LangChain”之前半段的对照物：上层 authoring 会越来越成熟，但跨系统长交易、能力对象和可重放审计仍是空位。

可行动建议与产出物：建议形成一页 MAF 1.10.0 vs ALUX 长交易运行时对照：Durable Task worker 对应可恢复执行，tool approval 对应能力授予，telemetry 对应可重放证据。 MAF 对比页 + Agent trace 字段映射草案。

RISC: R 主 · 强韧/机体次维度 S
持久执行是release 明确加入 standalone Durable Task worker 与 HarnessAgent loop。
故障恢复部分release 包含 fan-in checkpoint、workflow bugfix 与 streaming parent context 修复，但未证明跨组织恢复。
合作/学习窗口

适合面向 Microsoft/Foudry 生态做“MAF 负责 agent authoring，ALUX 负责跨组织长交易证据层”的互补叙事。

风险边界

风险是 Microsoft 会把 durable、approval、telemetry 都向 Azure/Foundry 闭环推进，压缩独立运行时的话语空间。

今天的新变化

今天观察到 1.10.0 的 durable worker、审批默认化和遥测修复，较前日 1.9.0 更接近生产运行时。

03OpenAI Codex美国/全球2026-06-25 / 2026-07-04官方研究

OpenAI 发布 Codex 经济研究：Agent 工作单位从一次回复转向分钟到小时级委派任务

发生了什么：OpenAI 报告显示 Codex 用户正在把更长、更复杂、跨职能的任务交给 agent；到 2026 年 5 月，80.6% 抽样个人用户至少提交过一次估计超过 30 分钟人类工作的请求。

对 ALUX 的关系：这是今天最高价值信号：当 Codex 运行时间被经济研究量化，ALUX 的“长交易”不再是技术想象，而是企业已经在消耗的 agentic labor。融资叙事可从“更聪明模型”转向“谁为长程 agent 劳动提供运行时责任链”。

可行动建议与产出物：建议把今日主叙事定为“Agentic labor needs runtime accountability”，并形成一页长交易工时证据图。 长交易工时证据图 + investor why-now 页。

RISC: R 主 · 强韧/机体次维度 C
持久执行部分来源证明用户把分钟到小时级任务交给 Codex，但未说明底层恢复机制。
水平扩展是报告覆盖 individual、organizational 与 OpenAI 内部跨部门使用，并提到多 agent 并行。
合作/学习窗口

可与 eval、observability、企业流程咨询方合作，把 agent 工时转化为 trace、成本、风险和恢复指标。

风险边界

报告数据来自 OpenAI 自身与 Codex 样本，不能直接外推到所有行业；对外使用需标明方向性。

今天的新变化

此前已观察 OpenAI agents work 趋势；今天选入是为了支撑“长程工作负载”已成经济研究对象。

04Anthropic Claude Sonnet 5美国/全球2026-06-30 / 2026-07-04官方发布

Anthropic Claude Sonnet 5 下放 agentic 执行能力，成本层开始承接浏览、终端和持续编码

发生了什么：Sonnet 5 被定位为最 agentic 的 Sonnet 模型，可规划、使用浏览器和终端，并在 Claude Code 与平台 API 中提供更低成本的持续执行能力。

对 ALUX 的关系：Sonnet 5 说明强大脑会继续降价并下放到更广可用层。ALUX 不应去比模型能力，而要利用这个趋势证明：当大脑更便宜、更普及，企业更缺机体、免疫和社会。

可行动建议与产出物：建议在 deck 中加入“大脑快速商品化，运行时责任稀缺化”页，Sonnet 5 作为证据之一。 大脑商品化 vs 运行时稀缺化叙事页。

RISC: I 主 · 智能/大脑次维度 R
模型循环是来源称 Sonnet 5 可 make plans、use browsers and terminals、run autonomously。
工具编排是浏览器和终端工具使用是发布核心。
合作/学习窗口

可把 ALUX 定位成 Claude/OpenAI/Qwen 等多模型的中立运行时，不依赖某一个大脑。

风险边界

不能把 Sonnet 5 的安全评估等同于企业级权限安全；模型更安全不代表工具调用边界自动安全。

今天的新变化

7 月初的关键变化是 Sonnet 级成本层接近 Opus 4.8 的部分 agent 能力，让“大脑商品化”更快。

05Amazon Bedrock AgentCore美国/全球云平台2026-06 / 2026-07-04官方文档

AWS Bedrock AgentCore June notes：Identity 引用既有 Secrets、Runtime 持久终端、Step Functions 嵌入 Harness

发生了什么：AgentCore 允许 Identity 引用既有 Secrets Manager ARN；Runtime 支持持久交互式 shell；Step Functions 可原生调用 AgentCore harness 并包裹人工审批、错误处理和条件路由。

对 ALUX 的关系：AWS 正把 agent 纳入既有云身份、Secrets、Step Functions 和 sandbox 体系，验证企业会先从治理与工作流接入购买 agent runtime。ALUX 的差异要放在跨云、跨公司、可重放证据。

可行动建议与产出物：建议补一张“Cloud AgentCore vs Neutral Agent Runtime”表格，突出云内治理与中立长交易的边界。 Cloud AgentCore vs Neutral Agent Runtime 表格。

RISC: S 主 · 安全/免疫次维度 R
能力对象部分AgentCore Identity 可引用 Secrets ARN 并用客户治理策略管理，但不是对象能力模型。
策略审批是Step Functions 可包裹 human approval、error handling 和 conditional routing。
合作/学习窗口

可服务不想被单一云锁定的客户，提供云外中立 runtime 证据层。

风险边界

AWS 拥有采购、IAM 和工作流入口；ALUX 不能只讲“也能审批”，要讲跨系统长交易和审计证明。

今天的新变化

今天重点不再是 Web Search，而是 Identity、持久终端与 Step Functions 把 agent 纳入云工作流。

08Kong AI Gateway / Langflow / Dify美国/安全生态2026-07-02 / 2026-07-04公司安全博客

Kong 把 Langflow / Dify 漏洞串成 AI Agent Gateway 需求，强调 compromised agent 的行动半径大于普通 Web 表单

发生了什么：Kong 文章将 Langflow RCE、Dify 跨租户数据暴露等风险归为结构性治理缺口，认为 agent 会带着授权调用工具和触发工作流，攻击半径更大。

对 ALUX 的关系：Kong 把 agent compromise 的 blast radius 讲清楚：Agent 不只泄露数据，还会带着权限行动。ALUX 可把这类 gateway 安全叙事推进一步，强调工具调用之后的长交易状态、能力衰减和可重放追责。

可行动建议与产出物：建议把 Runtime vs Gateway 对比页升级为销售资产：网关拦截请求，ALUX 记录授权、状态转移和回放证据。 Runtime vs Gateway 销售对比页。

RISC: S 主 · 安全/免疫次维度 C
隔离边界是文章指出 Dify 跨租户数据暴露和内部 API 未授权访问，隔离失败是核心风险。
策略审批部分Kong 主张 gateway 治理层，但文章不是具体策略审批产品说明。
合作/学习窗口

可和 API gateway / IAM / security vendors 合作，让 ALUX trace 成为它们拦截之后的证据层。

风险边界

Kong 等网关厂商会抢“Agent 安全入口”预算；ALUX 必须说明 gateway 管流量，ALUX 管长交易执行与证明。

今天观察到该信号与生产级运行时叙事的关系更清晰。

融资 / 合作窗口

最直接窗口：企业安全网关、云工作流、Agent observability、终端编码智能体、中国开源模型生态。它们都在处理同一件事：Agent 开始携带权限、状态和组织上下文行动。
融资叙事窗口：把 OpenAI 的工时研究、Claude Tag 的团队会话、MAF 的 durable worker、AWS 的 Identity/Step Functions 和 Kong 的漏洞叙事串起来，说明运行时责任链正在成为独立市场。

技术 / 产品启发

优先产品：RISC scorecard demo。每个 demo action 都给出主维度、两条证据、能力对象、状态推进、恢复点和回放判定。
优先 Demo：Slack/终端/云工作流混合长交易：团队频道发起、终端执行、Step Functions 审批、失败补偿、审计回放。

风险边界

ALUX 不能被说成已经完整交付智能体平台。准确说，当前底层 TVM 已具备并发、持久化执行、能力安全、运行记录和逐比特重放审计等关键基础；智能体产品层、可观测性、仪表盘、追踪与评估工具仍是待构建和融资重点。

也不要说 TVM 让 LLM 本身确定性。准确说，TVM 记录模型输出和运行环境输入，使编排、权限、状态转移和审计可重放、可验证。
