# ALUX 对外扫码卡标准

本标准用于生成 ALUX AI智能体情报日报的微信分享扫码卡。卡片是正式刊物的延伸，不是通用 SaaS 宣传海报。

## 固定交付

- 最终文件：JPG。
- 尺寸：3072×4096 像素，严格 3:4。
- 色彩模式：RGB。
- JPG 质量：95，4:4:4 采样。
- 对外只交付 JPG。渲染过程允许产生临时 PNG，但验证完成后必须自动删除。
- 本机成品统一输出到仓库根目录的 `output/`；该目录不提交 GitHub。
- 经确认的正式成品另存到 `assets/share-cards/` 并提交 GitHub；不提交中间稿或临时 PNG。
- 中文站二维码和公开网址固定为 `https://ai.alux.network/daily/`。
- 英文站公开地址固定为 `https://ai.alux.network/daily/en/`。
- `https://ai-agent-daily.alux.network/` 只作为永久兼容入口；已发布的旧扫码卡可继续使用，但新生成或重新导出的扫码卡不得再编码或展示旧域名。
- 不得使用 Vercel 预览域名制作对外物料。

## 版式与信息层级

- 逻辑画板固定为 768×1024 CSS 像素，使用 `deviceScaleFactor: 4` 渲染。
- 复用日报的纸张灰白、深海军蓝 `#14213d`、靛蓝 `#22356f`、青绿 `#0f766e` 与细分隔线。
- 中文主标题使用宋体链；正文与扫码指令使用系统无衬线；英文使用正式刊名 `ALUX AI Agent Intelligence Daily`。
- 只保留品牌、刊名、三项核心卖点、二维码、扫码指令和正式域名。
- 二维码必须是全图最大的视觉焦点。
- 采用“二维码主区 + 信息导轨”的编辑式分栏：二维码占主要面积，侧栏只承载扫码动作、三项核心特性和方向提示；不得让二维码居中后在深色面板内留下大块无信息空区。
- 上半部分使用刊名、内容覆盖范围和固定入口说明建立出版物层级；不写容易过期的具体期数、日期或统计数字。
- 禁止堆叠圆角卡片、胶囊标签、假链接输入框、重复英文眉题和底部微型说明。
- 所有重要内容距画布边缘至少 48 CSS 像素；底部文字不得贴边或被面板裁切。

## 二维码门槛

- 纠错级别：H。
- 黑色模块：`#000000`；背景：`#ffffff`。
- 四周静区至少 4 个模块。
- 静区内不得放 Logo、文字、纹理、边框、阴影、渐变或圆角。
- 不得旋转、抗锯齿、加中心 Logo 或改变模块形状。
- 黑色数据区目标宽度为画布的 44%–50%。
- 最终成品必须从整张卡片解码，不能只验证二维码源文件。

## 固定生成流程

在仓库根目录执行：

```powershell
python .\tools\share-card\generate_qr.py
node .\tools\share-card\render_share_card.cjs
python .\tools\share-card\finalize_share_card.py
```

生成逻辑必须等待字体与图片加载完成，检查 768×1024 画板没有滚动或文字越界，再输出 JPG。`finalize_share_card.py` 会验证 3072×4096 RGB 元数据，并在 4K、1080、720、540、360 像素宽的 JPEG 压缩模拟中解码二维码；任一测试失败都不得交付。

## 禁止回归

- 不得为了“看起来丰富”增加重复信息。
- 不得回退到只有居中二维码和大面积空白底板的构图；信息密度必须来自真实阅读入口和内容结构，不得使用无意义装饰填空。
- 不得缩小二维码来给装饰让位。
- 不得依赖图片查看器裁图、浏览器整页截图或后期拉伸取得 3:4。
- 不得保留最终 PNG 成品或把临时 PNG 提交到 GitHub。
- 不得只修改导出图片而不更新模板、生成脚本和本标准。
