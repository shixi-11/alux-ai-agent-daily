# ALUX 双语响应式与防重叠标准

本规范用于防止中文模板换成英文后出现跨栏、遮挡、裁切或靠缩小字号勉强塞入的问题。所有维护者和自动化 Agent 在生成、翻译、调整公共样式或发布前都必须遵守。

## 硬性结果

- 在 `1920、1440、1024、768、620、430、390、320px` 下，所有可见文字、徽章、图标和控件必须留在所属的 grid/flex 单元内。
- 非祖先关系的相邻元素不得相交；“没有整页横向滚动”不等于没有内部重叠。
- 长英文标签、破折号复合词、产品名和 URL 必须通过自然换行或断点改单列解决。
- 报告页的 `.panel-head` 标题与右侧说明各自留在所属单元内；`920px` 及以下改为上下排列，禁止标题压住右侧说明。
- 顶栏 Logo 可见外框与语言切换可见外框统一为 `44px` 高并垂直居中；每个语言按钮的点击高度不得低于 `44px`。
- 不得用 `overflow: hidden` 掩盖、手工插入固定 `<br>`、缩到难以阅读，或改变英文原意来换取视觉通过。

## 英文热区矩阵的固定方案

中文母稿内嵌的 `68–76px` 标签列只适合短中文，英文页面由 `assets/report-site.css` 统一覆盖：

- `621px` 以上：`.heat-row` 左栏至少 `172px`，右栏使用 `minmax(0, 1fr)`，列间距 `18px`。
- `.heat-row strong` 与 `.strength` 允许在左栏内自然换行，禁止 `nowrap` 把文字挤入正文列。
- `620px` 及以下：改为单列；标签和强度徽章可同行换行，正文另起一行。
- 修复必须进入共享源样式，覆盖全部历史页和未来新一期；不得只改某个 `public/en/YYYY/MM/DD/index.html`。

## 自动验收门禁

`scripts/render-check.cjs` 必须：

1. 抽查中英首页和最早、中间、最新日期页的 8 档视口。
2. 对全部英文历史页完整执行 `1920、1440、1024、768、620、430、390、320px` 八档检查，并额外检查 `621px`、`920px`、`921px` 三个断点边界。
3. 使用浏览器 `Range#getClientRects()` 读取真实文字边界；只比较元素外框会漏掉文字已经越栏而 block 外框仍未变化的错误。
4. 同时检查 `.heat-row` 标签/徽章、`.panel-head` 标题/说明、整页横向溢出、可视元素裁切，以及 Logo 与语言切换外框高度。
5. 固定保留 `2026-07-15` 作为英文长标签回归页。
6. 当文字越过所属单元、与非祖先相邻元素相交或顶栏控件错位时，以非零状态退出，阻止提交和推送。

不得删除、跳过或降低这项门禁。若门禁失败，按“共享 CSS → 重新生成 → 结构验证 → 真实 Chrome 渲染”的顺序修复。

## Agent 修正流程

1. 从 `content/` 和共享样式定位问题，不直接编辑 `public/`。
2. 优先使用 `min-width: 0`、`minmax(0, 1fr)`、合理英文列宽、自然换行和断点重组。
3. 运行 `scripts/sync-reports.ps1` 与 `scripts/verify-site.ps1`。
4. 运行 `node scripts/render-check.cjs`，确认全部英文历史页的专项检查通过。
5. 人工查看最新英文页在桌面、1024px、620px 和手机宽度下的热区矩阵、面板标题与顶栏控件。
6. 只有本地与正式域名均验收通过后才能宣布完成。
