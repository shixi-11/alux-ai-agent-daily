# ALUX AI智能体情报日报风格规范

## 刊物定位

这是一份面向 AI Agent 基础设施与生产化的每日情报刊物。它不只罗列新闻，还要判断外部信号对可靠执行、恢复、权限、审计、长交易和跨组织协作的意义，并严格区分“行业已经交付”、“可借鉴”和“ALUX 尚待构建”。

## 编辑原则

- 以官方发布、官方 GitHub、官方技术博客、可验证论文和可靠商业来源为主。
- 标题写“发生了什么变化”，不写空泛的趋势词。
- 每条重点信号回答四个问题：发生了什么；与 ALUX 有什么关系；能产出什么具体资产；证据边界在哪里。
- 不把融资公告当成技术证明，不把 trace、log 或 system of record 直接写成可重放执行证据。
- 不宣称 ALUX 已经完整交付智能体平台。准确说法是：底层 TVM 已具备并发、持久化执行、能力安全、运行记录与逐比特重放审计等关键基础；智能体产品层、可观测性、仪表盘、追踪和评估工具仍待构建。
- 不说 TVM 让 LLM 本身确定性。应说 TVM 记录模型输出与运行环境输入，使编排、权限、状态转移与审计可重放、可验证。

## 每期固定结构

1. **Hero**：日期、情报类型、主标题、一段 lead、四项统计与“今日总判断”。
2. **今日热区矩阵**：用 RISC 四维展示今日强弱与 ALUX 动作。
3. **RISC 机器说明**：在新版日报中保留 R｜强韧/机体、I｜智能/大脑、S｜安全/免疫、C｜连接/社会的简明定义。
4. **ALUX 今日雷达**：机会、风险、可行动资产各一张。
5. **重点信号**：每条包含 meta、标题、“发生了什么”、“对 ALUX 的关系”、“可行动建议与产出物”、RISC 证据卡与侧栏风险边界。
6. **融资/合作窗口**：只写有对应证据的合作对象与资本叙事。
7. **技术/产品启发**：给出一个优先产品或 schema，以及一个可演示的 Demo。
8. **风险边界**：明确什么不能从当日证据推导，以及 ALUX 当前真实能力与待建层。
9. **来源**：与重点信号逐项对应，保留原始链接和来源级别。

## 中文文风

- 自然、顺、有人味，但保持情报刊物的准确和节制。
- 一段只承担一个完整意思，不为节奏把正常语法拆成碎句。
- 中英文、数字与单位之间的空格保持一致；中文标点前不加空格。
- 站名固定为 `ALUX AI智能体情报日报`。
- 避免“其实”、泛化的“赋能”、“颠覆”、“革命性”与无证据的“首个”、“全球领先”。

## 英文文风

- 使用美式英语，写成原生的 technology-intelligence publication，不保留中文句式。
- 标题简洁、有动作；正文先讲证据，再讲判断与行动。
- 术语、大小写与刊名以项目翻译配置为准。
- 定稿前必须完成分析、初译、审校和润色；不使用未复核的逐句机翻。

## 视觉与响应式

- 气质：正式编辑刊物，冷静、精密、可信。
- 颜色：纸张灰白、深海军蓝、青绿辅助色和低饱和琥珀；不默认使用红色。
- 字体：大标题使用端正衬线，正文使用高可读无衬线。
- 构图：桌面端可用双栏首屏，平板和手机必须收为单栏。所有网格、长英文专名、来源 URL 和 RISC 卡片必须在 320 px 下可读且无裁切。
- 品牌：使用 `assets/alux-mark.png` 作为浏览器图标、首页品牌标识和日报页站点栏标识。

### 英文长标签防重叠

- 中文热区矩阵的 `68–76px` 标签列只适合短中文，不得直接用于英文。
- 英文 `.heat-row` 在 `621px` 以上使用至少 `172px` 的标签列，右栏必须保持 `minmax(0, 1fr)`；标签列与正文列之间保留明确间距。
- `.heat-row strong` 和 `.strength` 必须允许在自身单元内自然换行，不得用 `white-space: nowrap` 把文字挤入右栏。
- `620px` 及以下将英文热区矩阵改为单列，标签与强度徽章可以同行换行，正文另起一行。
- `.panel-head` 的英文标题与右侧说明必须保留独立空间，`920px` 及以下改为上下排列。
- 顶栏 Logo 可见外框和语言切换可见外框统一为 `44px` 高并垂直居中，语言按钮的独立点击高度不低于 `44px`。
- 自动验收使用真实文字 Range 检查跨栏和相交；不得用隐藏溢出、固定换行、缩小字号或改写译文掩盖布局问题。
- 禁止只凭“页面没有横向滚动”判定合格。必须运行 `scripts/render-check.cjs`，确认标签或徽章的渲染边界没有越过左栏、与右栏正文相交。
- 这类修复必须进入 `assets/report-site.css` 和自动化验收脚本，不能只改某一期生成后的 `public/` 页面。

## 创建新一期时的版式基准

以 `content/zh/` 中日期最新、已通过渲染验收的日报作为 HTML 结构骨架，保留类名与响应式规则，但不复制旧内容、旧数字或旧判断。如果调整公共页面风格，同步修改模板或 `assets/report-site.css`，不只修当期的一个页面。
