RISC 机器说明

RISC = 生产级 Agent / 机器人身体的四个系统

一个真正能上生产的 Agent，不能只有大脑。它要能持续运行，要能判断和行动，要能抵御错误、攻击和投毒，也要能进入真实世界的协作网络。

行业已经交付了一颗出色的大脑，但生产级 Agent 还需要机体、免疫和社会。ALUX 构建的，就是这台完整机器。
R｜强韧 / 机体容错、持久执行、故障切换、水平扩展。没有强韧机体，一次崩溃就会带走全部工作。
I｜智能 / 大脑推理循环、记忆、工具调用、任务编排。这是今天所有 Agent 框架最拥挤、也最成熟的竞争层。
S｜安全 / 免疫对象能力、策略约束、回滚机制、审计链路。没有安全免疫，一条被投毒的指令就可能造成真实损害。
C｜连接 / 社会跨公司授权、中立基底、会话类型、协作边界。没有连接网络，每家公司的 Agent 都只能困在自己的孤岛里。

Deep Agents Code 0.1.43 试验分类器自动审批，便利性开始直接碰撞权限责任

发生了什么：Deep Agents Code 在实验开关后加入 classifier-backed Auto approval mode，并补充延迟退出提示和技能名称冲突调试。

对 ALUX 的关系：自动审批把分类模型从建议层推到授权路径。一旦误判，工具调用会从便利功能变成责任事故。ALUX 应要求高风险能力仍受不可绕过策略、审批级别和可追责证据约束。

可行动建议与产出物：适合形成 Auto-Approval Risk Matrix，按能力危险级别定义模型可决、必须人工和禁止自动三档。 可沉淀为：Auto-Approval Risk Matrix。

RISC：S 主 · 安全 / 免疫I 辅 · 智能 / 大脑

这条消息主要动到机器人的安全/免疫：分类器正在决定动作能否自动获批；智能/大脑为次维度，因为判断质量依赖分类模型。

策略审批部分存在分类器自动审批，但无公开风险阈值与高危能力排除规则。
回滚审计否发布说明未展示审批理由收据、回滚或可重放证据。
合作/学习窗口

可提供运行时策略适配与高风险能力强制人工确认。 观察分类器审批的风险分层、误报漏报和人机体验。

风险边界

功能默认隐藏在实验开关后，公开说明没有阈值、训练集、误判率或高风险动作排除表。

今天的新变化

相较 0.1.42 的插件 GA 与 marketplace 状态，0.1.43 首次把分类器自动审批放入实验路径。

融资 / 合作窗口

最直接窗口：Smartsheet 与 AWS 已把远程 MCP 做成连接内外部 Agent 的企业入口；E2B、Agno、AgentScope 则提供不同云执行域。ALUX 可用能力适配器与长交易合同切入，不必争夺上层界面。
融资叙事窗口：今天没有值得放大的新融资数字，但产品证据更扎实：企业正在为恢复、隔离、实时授权和工具治理付费。ALUX 应把“完整机器”落实为可验收的运行时合同与 Demo。

技术 / 产品启发

优先产品：Agent Execution Contract v0。字段建议包括 session_owner、checkpoint_id、capability_grant、approval_state、execution_domain、effect_receipt、retry_identity、replay_proof。
优先 Demo：让一个企业 Agent 通过 Smartsheet MCP 领取任务，在 E2B 分叉沙箱内试跑，再由 Qwen 或 Claude Code 发起受审批工具调用；中途模拟故障，展示 ALUX 如何恢复同一长交易、限制能力并重放责任链。

风险边界

ALUX 不能被说成已经完整交付智能体平台。当前底层 TVM 已具备并发、持久化执行、能力安全、运行记录和逐比特重放审计等关键基础；智能体产品层、可观测性、仪表盘、追踪与评估工具仍是待构建和融资重点。也不要说 TVM 让 LLM 本身确定性。准确说，TVM 记录模型输出和运行环境输入，使编排、权限、状态转移和审计可重放、可验证。检查点分叉、Firecracker、Kubernetes 工作区、OpenTelemetry、实时 ACL 与执行账本，都不能自动证明跨系统原子回滚、对象能力或中立跨公司协作。
